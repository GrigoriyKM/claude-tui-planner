# База данных

SQLite-хранилище Quest с WAL-режимом и автоматической инициализацией схемы.

## Когда читать
При добавлении новой таблицы или колонки, отладке миграций, изменении логики подключения.

---

## Ключевые компоненты

- `quest/db.py` — подключение, схема, инкрементальные миграции
- `data/quest.db` — файл БД (`~/.claude/quest/data/quest.db`)

---

## Таблицы

| Таблица | Описание |
|---------|---------|
| `tasks` | Задачи с размером, статусом, XP, датами, приоритетом |
| `daily_logs` | Ежедневные итоги, стрик-флаги, оценки дня |
| `streaks` | Единственная строка с текущим и максимальным стриком |
| `user_stats` | Единственная строка с уровнем и суммарным XP |
| `persistent_notes` | Единственная строка (`id=1`) с текстом панели заметок |

---

## Миграции

Схема применяется через `run_migration(conn, SCHEMA_SQL)` — идемпотентные `CREATE TABLE IF NOT EXISTS`.

Инкрементальные миграции (колонки/таблицы, добавленные после выпуска) — в `_apply_incremental_migrations()`:

| Миграция | Условие проверки |
|----------|-----------------|
| `priority` колонка в `tasks` | `PRAGMA table_info(tasks)` |
| Таблица `persistent_notes` | `sqlite_master WHERE type='table'` |

Добавляя новую миграцию — следуй тому же паттерну: проверь существование, примени, `commit()`.

---

## Конфигурация подключения

- **WAL mode** — параллельные читатели не блокируют запись
- **foreign_keys=ON** — `parent_id → tasks.id` проверяется на уровне БД
- `row_factory = sqlite3.Row` — доступ к колонкам по имени

---

## Поток данных

1. `init_db()` — создаёт файл и таблицы через `SCHEMA_SQL`, затем применяет инкрементальные миграции
2. `get_db()` — открывает соединение с WAL и `foreign_keys=ON`
3. `db_exists()` — проверяет наличие файла БД (используется TUI для graceful no-DB режима)

---

## Инициализация

```bash
cd ~/.claude/quest && uv run python scripts/quest_cli.py init
```

Сидирует `user_stats (id=1)` и `streaks (id=1)` через `INSERT OR IGNORE`.
