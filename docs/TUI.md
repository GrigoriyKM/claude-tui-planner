# TUI-интерфейс

Textual-приложение для интерактивной работы с задачами в терминале.

## Когда читать
При добавлении новых экранов или виджетов, изменении горячих клавиш, отладке фокуса или навигации.

---

## Компоненты

| Файл | Роль |
|------|------|
| `tui/app.py` | Главное приложение `QuestApp` — биндинги, экшены, координация |
| `tui/widgets/stats_panel.py` | Панель уровня, стрика и XP-бара |
| `tui/widgets/task_list.py` | Прокручиваемый список задач с курсором |
| `tui/widgets/notes_panel.py` | Боковая панель заметок с debounced-автосохранением |
| `tui/screens/add_task_screen.py` | Модальный экран добавления задачи (`RichAddTaskScreen`) |
| `tui/screens/edit_task_screen.py` | Модальный экран редактирования (`RichEditTaskScreen`) |
| `tui/screens/snooze_task_screen.py` | Модальный экран откладывания задачи (`SnoozeTaskScreen`) |
| `tui/__main__.py` | Точка входа для `python -m tui` |

## Раскладка экрана

```
┌─────────────────────────────────────────┬──────────────┐
│ StatsPanel: Level · 🔥 streak · XP bar  │              │
│ Filter bar: all | today | overdue | done│  NotesPanel  │
│                                         │  (30%, n/N)  │
│ TaskListWidget (прокручиваемый список)  │              │
│   TODAY                                 │              │
│   [ ] Task A   small   50 XP            │              │
│ > [ ] Task B   medium  150 XP           │              │
│                                         │              │
│ Footer (горячие клавиши)                │              │
└─────────────────────────────────────────┴──────────────┘
```

`NotesPanel` скрыт по умолчанию (CSS-класс `.hidden`), появляется по `n`.

## Горячие клавиши

| Клавиша | Действие |
|---------|---------|
| `j` / `↓` | Вниз по списку |
| `k` / `↑` | Вверх по списку |
| `ctrl+d` / `ctrl+u` | Полстраницы вниз / вверх |
| `g` / `G` | Первая / последняя задача |
| `Tab` / `l` / `→` | Сменить фильтр вперёд |
| `S-Tab` / `h` / `←` | Сменить фильтр назад |
| `[` | На вкладке today: переключить завтра; на done: следующий диапазон |
| `Enter` / `Space` | Выполнить / отменить выполнение |
| `a` | Добавить задачу (модальный экран) |
| `e` | Редактировать задачу (модальный экран) |
| `z` | Отложить задачу (snooze, модальный экран) |
| `d` `d` | Удалить задачу (двойное нажатие, 1.5 с таймаут) |
| `i` | Инспектировать задачу (overlay с деталями) |
| `y` | Скопировать название задачи в буфер обмена |
| `r` | Обновить данные из БД |
| `n` | Переключить панель заметок |
| `N` | Переключить фокус в панель заметок (Esc — назад) |
| `?` | Справка |
| `q` | Закрыть overlay / выйти |

## Модальные экраны

### `SnoozeTaskScreen`
Принимает `Task`, возвращает `str | None` (дата `YYYY-MM-DD`).
Поля: `RadioSet` с пресетами (завтра / 7 / 14 дней) + `Input` для произвольной даты.
Ввод в `Input` переопределяет `RadioSet`. Валидация: дата должна быть в будущем.

### `RichAddTaskScreen` / `RichEditTaskScreen`
Возвращают `AddTaskResult | None`. При открытии на вкладке tomorrow — устанавливают `due_date` на завтра.

### Поведение Tab/S-Tab в модалах
`action_cycle_filter` проверяет `_modal_active()` и перенаправляет `Tab`/`S-Tab` на `focus_next()` / `focus_previous()` экрана, не меняя фильтр.

---

## Удаление с подтверждением

`d` вооружает удаление (`_delete_armed_task`). Второй `d` на той же задаче — выполняет удаление. Таймер 1.5 с сбрасывает состояние. Любое перемещение курсора (`j`, `k`, `ctrl+d`, `g`, `G`) тоже разоружает.

---

## Панель заметок

`NotesPanel` хранит текст в таблице `persistent_notes` (одна строка, `id=1`).
Автосохранение: debounce 2 с после изменения `TextArea`. При `on_unmount` — немедленный flush.
Фокус: `n` — показать панель, фокус остаётся на списке; `N` — перевести фокус в `TextArea`; `Esc` — вернуть фокус на список.

---

## Поток данных

1. `on_mount` → `_open_db()` → `_load_all()`
2. `_load_all()` — читает задачи по активному фильтру, обновляет `StatsPanel`, `filter-bar`, `TaskListWidget`
3. Любое изменение (toggle / add / edit / snooze / delete) → `_load_all()`
4. `TaskListWidget.load_data()` → `_refresh_display()` → `_scroll_to_cursor()` (cursor остаётся виден)
